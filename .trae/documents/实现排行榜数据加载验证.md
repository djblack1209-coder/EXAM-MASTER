# 实现排行榜数据加载验证

## 目标
确保排行榜页面能正确调用云函数/API，获取并展示用户排名数据。

## 问题分析
1. 当前排行榜页面使用的是模拟数据（mock data），没有从后端获取真实数据
2. 需要调用 `rank-center` 云函数（已迁移到 Sealos）来获取排行榜数据
3. 需要确保数据结构符合预期，包含 `list` 和 `myRank` 字段
4. 需要确保 UI 展示正常，包括前三名、列表区域和底部固定栏

## 实现步骤

### 1. 修改排行榜页面，添加真实数据加载逻辑
- 在 `pages/practice/rank.vue` 中引入 `lafService`
- 添加数据加载方法 `loadRankData()`，调用 `lafService.rankCenter()` 获取排行榜数据
- 在 `onLoad()` 生命周期中调用 `loadRankData()`

### 2. 调整数据结构，适配后端返回数据
- 修改 `rankList` 和 `otherRanks` 数据结构，适配后端返回的 `list` 字段
- 添加 `myRank` 数据处理逻辑，适配后端返回的 `myRank` 字段
- 确保列表项包含：`nickname`、`avatar`、`score` 等字段

### 3. 优化 UI 展示，确保符合预期
- 确保前三名（Top 3）有特殊的皇冠/领奖台样式
- 确保列表区域显示第 4 名及以后的用户
- 确保底部固定栏显示当前登录用户的排名和分数

### 4. 添加异常处理和空状态
- 添加数据加载失败的处理逻辑
- 添加数据库为空时的空状态展示
- 添加加载动画，提升用户体验

### 5. 验证 API 调用和数据结构
- 确保控制台显示调用了 `rank-center` 云函数
- 确保返回的数据包含 `list` 和 `myRank` 字段
- 确保列表项包含预期的字段

## 技术要点
- 使用 `lafService.rankCenter()` 调用后端 API
- 数据结构：返回数据应包含 `list` 和 `myRank` 字段
- UI 展示：前三名有特殊样式，列表区域和底部固定栏正常显示
- 异常处理：处理数据加载失败和空状态
