# Exam-Master v2.0 开发规划

## 一、技术架构升级

### 1. 前端架构升级
- **核心框架**：保持 UniApp + Vue3 + Pinia 不变
- **UI 渲染优化**：
  - 将 @qiun/ucharts 升级为支持 WebGL 的渲染库（如 Three.js 小程序适配版）
  - 优化 Canvas 渲染性能，支持 3D 星空流畅交互
- **状态管理增强**：
  - 引入 Pinia 持久化插件
  - 优化跨页面状态共享
- **网络通信优化**：
  - 升级 WebSocket 支持，实现云自习室实时状态同步
  - 优化 API 请求策略，实现智能缓存

### 2. 后端架构升级
- **AI 服务层重构**：
  - 建立 Prompt 管理系统，支持不同学科的评分标准
  - 集成向量数据库，实现 AI 长期记忆功能
  - 优化 LLM 调用策略，提高响应速度
- **OCR 服务集成**：
  - 对接第三方 OCR 服务（如百度 OCR、腾讯 OCR）
  - 实现图像压缩与预处理，提高识别准确率
- **WebSocket 服务优化**：
  - 评估 Laf WebSocket 承载能力，必要时引入独立 Socket.io 服务

### 3. 数据架构升级
- **知识图谱数据模型**：
  - 设计学科知识图谱结构
  - 实现知识点与题目、讲义的关联映射
- **用户行为数据模型**：
  - 扩展用户学习行为数据采集
  - 实现学习进度与知识图谱的关联

## 二、功能模块开发计划

### 1. Sprint 1（基础建设 - 2周）

#### 1.1 OCR 拍照搜题/录题
- **前端**：
  - 集成 uni-camera 组件
  - 实现图像压缩与预处理
  - 开发拍照界面与交互
- **后端**：
  - 对接 OCR 服务 API
  - 实现题目识别与解析
  - 开发题目入库接口

#### 1.2 向量数据库搭建
- **后端**：
  - 集成向量数据库（如 Laf 集成的 pgvector）
  - 实现用户学习历史向量化存储
  - 开发向量检索接口

#### 1.3 UI 组件库升级
- **前端**：
  - 新增专注模式相关组件
  - 优化情感化设计元素
  - 实现微交互与粒子特效

### 2. Sprint 2（核心功能 - 3周）

#### 2.1 知识星空 v2.0 - "星图导航"
- **前端**：
  - 集成 Three.js 实现 3D 星空渲染
  - 开发知识图谱可视化界面
  - 实现知识点状态映射（黯淡/点亮/预警）
  - 开发星星点击跳转功能
- **后端**：
  - 设计知识图谱数据结构
  - 实现知识点与题目关联
  - 开发知识图谱查询接口

#### 2.2 沉浸式云自习室
- **前端**：
  - 实现番茄钟功能
  - 集成白噪音播放
  - 开发虚拟同桌界面
  - 实现状态同步与检测
  - 开发专注报告生成功能
- **后端**：
  - 实现 WebSocket 状态同步
  - 开发用户在线状态管理
  - 实现专注报告数据接口

#### 2.3 AI 智能批改
- **前端**：
  - 开发主观题拍照上传界面
  - 实现批改结果展示
  - 开发逐句修改建议展示
- **后端**：
  - 实现 OCR 识别主观题
  - 开发 AI 批改逻辑
  - 实现评分标准管理

### 3. Sprint 3（优化与发布 - 1周）

  - 实现遗忘曲线算法

  - 实现情绪安抚功能
- **前端**：
  - 开发助教消息推送界面
  - 实现情绪反馈交互

#### 3.2 性能优化与测试
- **前端**：
  - 优化 Canvas 渲染性能
  - 测试 WebSocket 连接稳定性
  - 实现暗黑模式优化
- **后端**：
  - 优化 API 响应速度
  - 测试 OCR 识别准确率
  - 测试 AI 批改效果

#### 3.3 灰度发布
- 实现灰度发布策略
- 收集用户反馈
- 修复上线问题

## 三、前后端协作方案

1. **API 设计规范**：
   - 采用 RESTful API 设计风格
   - 统一错误码格式
   - 实现 API 版本管理

2. **数据交互协议**：
   - 前端请求格式：JSON
   - 后端响应格式：统一包裹 { code, message, data }
   - WebSocket 消息格式：JSON

3. **联调测试方案**：
   - 前端使用 Mock 数据开发
   - 后端提供 API 文档（如 Swagger）
   - 定期进行前后端联调

4. **代码管理规范**：
   - 采用 Git Flow 工作流
   - 定期代码 Review
   - 统一代码风格（ESLint + Prettier）

## 六、风险与应对策略

1. **技术风险**：
   - Three.js 小程序适配问题：提前调研适配方案，预留调试时间
   - WebSocket 性能问题：评估 Laf 承载能力，必要时引入独立服务
   - OCR 识别准确率问题：对接多个 OCR 服务，实现 fallback 机制

2. **时间风险**：
   - 功能开发延期：采用 MVP 策略，优先实现核心功能
   - 联调测试时间不足：提前规划联调时间，并行开发与测试

3. **质量风险**：
   - AI 批改效果不佳：持续优化 Prompt 工程，收集用户反馈迭代
   - 3D 渲染性能问题：优化渲染逻辑，实现分级渲染

## 七、团队协作建议

1. **角色分工**：
   - 前端团队：分为 UI 组、功能组、性能优化组
   - 后端团队：分为 API 组、AI 服务组、WebSocket 组
   - 测试团队：分为功能测试组、性能测试组

2. **沟通机制**：
   - 每日站会：同步进度与问题
   - 周会：总结本周工作，规划下周任务
   - 技术分享：定期分享技术难点与解决方案

3. **工具支持**：
   - 项目管理：使用 Jira 或 Trello 管理任务
   - 文档管理：使用 Confluence 管理技术文档
   - 即时通讯：使用企业微信或 Slack 沟通

## 八、总结

Exam-Master v2.0 版本将从"工具"升级为"智能伴侣"，重点关注用户的学习效率、情感陪伴与个性化体验。通过引入 OCR、3D 可视化、WebSocket 等技术，结合 AI Agent 与情感化设计，打造一个真正符合考研用户需求的智能备考平台。

开发团队需要密切协作，关注技术难点，确保项目按时高质量交付。同时，要持续收集用户反馈，不断优化产品体验，实现产品的持续演进。
