## 八、架构级建议

1. **认证中间件统一化**：当前每个云函数独立实现 JWT 校验，应提取为统一中间件，默认所有操作需鉴权
2. **前后端契约管理**：引入 API Schema（如 OpenAPI/Zod），自动校验请求/响应格式，避免字段名不匹配
3. **Serverless 状态管理**：内存态缓存/限流在 Serverless 环境无效，需迁移至 Redis 或数据库
4. **客户端信任边界**：所有涉及积分、金币、排名的计算必须在服务端完成，客户端仅做展示
5. **输入消毒统一化**：提取统一的 sanitize 中间件，所有用户输入在入口处统一处理
6. **定时器生命周期管理**：建立组件级定时器注册表，在 onUnload/onUnmounted 时统一清理
