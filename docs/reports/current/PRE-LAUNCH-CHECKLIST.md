# 🚨 上线前关键问题与建议

## 一、安全问题（已识别）

### P0: 客户端密钥暴露

**问题**: `VITE_OBFUSCATION_KEY` 和 `VITE_REQUEST_SIGN_SALT` 被打包到小程序代码中

**现状**:

- 这是 Vite 的预期行为（所有 VITE\_ 变量都会内联）
- 这些密钥用于"混淆"而非"加密"
- 客户端加密本质上无法防止逆向

**建议**:
✅ **可接受风险上线**，但需要：

1. 服务端不依赖客户端签名做安全验证
2. 重要操作必须有服务端二次验证
3. 敏感数据不存储在客户端

**长期方案**:

- 将加密逻辑移到服务端
- 使用动态密钥（服务端下发）
- 实现设备指纹 + 服务端验证
