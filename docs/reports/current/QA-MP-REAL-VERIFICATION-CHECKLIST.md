# EXAM-MASTER 真机验收清单（重点：刷题页 / 择校页 / 工具页）

适用环境：微信开发者工具 + 真机预览（建议 iOS + Android 各 1 台）。

## 0. 预置条件

- 使用已登录账号（确保有 `EXAM_USER_ID` 和 `EXAM_TOKEN`）。
- 网络正常（Wi-Fi）与断网（飞行模式）两种场景都要覆盖。
- 当前版本默认不依赖 `WechatSI` 插件（语音识别走后端 STT）。
- 若模拟器提示 `requiredPrivateInfos` 字段非法，先确认使用最新代码（已移除不兼容声明）再重新编译。
- 测试文件：
  - PDF：`【李彬】基层自治制度考点笔记.pdf`
  - PNG：`预览图0.png`

## 1. 刷题页（`/pages/practice/index`）

### 1.1 页面加载与关键入口

1. 打开刷题页，确认页面在 2 秒内可交互，无长时间“功能加载中”。
2. 点击“开始刷题”“错题本”“收藏夹”“AI 导师”“排行榜”等入口，确认都可跳转。
3. 返回刷题页后，入口仍可再次点击，不出现失效或卡死。

### 1.2 动态分包相关（高优先）

1. 首次冷启动直接进入刷题页，连续点击“开始刷题”。
2. 期望：若分包加载未完成，应短暂等待后正常进入，不应长期停留“功能加载中”。
3. 断网后重进刷题页再点击入口，期望有明确失败提示，不出现白屏。

### 1.3 本地搜索题库

1. 从拍照搜题回流到刷题页后，检查是否自动命中题库关键词并提示命中数量。
2. 点击进入搜索结果练习，确认模式为搜索练习（`mode=search`）。

## 2. 择校页（`/pages/school/index`）

### 2.1 三步流程

1. 第一步填写背景信息，检查必填校验（空值、非法字符）。
2. 第二步填写报考信息，使用带括号院校名（如“北京大学(医学部)”）提交。
3. 第三步查看结果，确认能正常展示推荐或空状态，不出现 loading 不消失。

### 2.2 AI 返回兼容

1. 测试返回 JSON 字符串与对象两种情况（后端不同版本）。
2. 期望：都能被解析并渲染；解析失败时进入空状态并关闭 loading。

### 2.3 目标院校状态一致性

1. 在择校页添加目标院校，返回列表后应显示“已在目标库”。
2. 删除目标院校后，列表状态应同步回“+ 加入目标”。
3. 重新打开择校页，状态应与 `target_schools` 持久化一致。

### 2.4 离线与缓存

1. 断网进入择校页：应优先展示缓存数据；无缓存时展示示例数据。
2. 恢复网络后应提示“网络已恢复”，可重新刷新推荐院校。

## 3. 文档转换（`/pages/tools/doc-convert`）

### 3.1 文件选择与校验

1. 切换 `PDF->Word`，选择 PDF 文件，确认可进入读取并可提交。
2. 切换 `图片->PDF`，选择 PNG 文件，确认可提交。
3. 选择不匹配扩展名时应前置拦截并提示。
4. 文件读取中按钮应显示“文件读取中...”，不可误提交。
5. 若控制台出现 `errno:112`（隐私协议 scope 未声明），页面应弹出“文件权限未开启”指引，不可无响应。

### 3.2 提交与结果

1. 点击开始转换后，观察状态：上传中 -> 转换中 -> 完成。
2. 结果链接字段兼容 `downloadUrl / url / files[0].url`。
3. 下载结果后在微信内可打开文档。

## 4. 证件照（`/pages/tools/id-photo`）

### 4.1 处理流程

1. 上传人像后选择背景色与尺寸，点击“开始制作”。
2. 期望：后端若返回 `needComposite=true`，前端可自动合成成品图。
3. 结果页可切换颜色重新生成，不丢图。

### 4.2 保存流程

1. 保存到相册成功时提示“已保存到相册”。
2. 拒绝相册权限时弹窗引导进入设置。
3. `data:image` 与 `wxfile://` 两种结果源都可保存。

## 5. 通过标准

- P0：无白屏、无卡死、无长时间 loading 不消失。
- P0：刷题页动态入口可稳定使用，择校提交后可进入结果页。
- P1：文档转换与证件照流程在主流机型可闭环完成。
- P1：错误场景提示明确，不出现静默失败。
- P0：`chooseMessageFile` 隐私未声明场景必须有可见弹窗引导，不可“点击无反应”。
