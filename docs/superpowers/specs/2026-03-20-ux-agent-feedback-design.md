# Spec: UX Agent Feedback & Flow State Integration

## 1. Overview
The backend `answer-submit` endpoint now successfully processes FSRS parameters and uses `ai-agent-team` to generate a Socratic tutor response. However, the frontend (`do-quiz.vue`) does not display this powerful data, falling back to basic "correct/incorrect" alerts. We need to expose the "Memory Engine Moat" to the user with a premium "Flow State" UX.

## 2. Approach
We will upgrade the post-answer experience in `do-quiz.vue` by building a custom Result Panel that handles two major new data streams:
1. **Agent Tutor Feedback**: When a user answers incorrectly, we will display the Socratic hint generated by the backend using a typewriter effect to simulate real-time AI thinking.
2. **FSRS Stats Visualizer**: When a user answers correctly (or incorrectly), we will display the updated memory parameters (e.g., "Next Review: 3 days", "Stability: +1.2").

## 3. Architecture Details
- **Data Layer**: Modify `do-quiz.vue`'s `handleSubmit` response handler to extract `memoryState` and `tutorFeedback` from the new backend response signature.
- **UI Components**:
  - `TutorFeedbackCard.vue`: A new business component using `<wd-card>` / `<BaseCard>` that wraps the Socratic feedback with the existing `useTypewriter` composable.
  - `MemoryStatsRow.vue`: A small, elegant row component displaying the `scheduled_days` and `stability` shifts.
- **Integration**: Inject these components into the `result-panel` section of `do-quiz.vue`.

## 4. Execution Plan
1. Create `TutorFeedbackCard.vue`.
2. Create `MemoryStatsRow.vue`.
3. Update `do-quiz.vue` state and UI to render the new components when a submission returns `data.tutorFeedback` or `data.memoryState`.
