# 文档摘要缓存 — 使用说明

> 本目录存放从官方文档中提炼的**关键结论**，是 AI 开发时的"速查手册"。

## 目的

避免每次都在线拉取完整文档页面（浪费上下文窗口），把已经验证过的关键信息存在本地。

## 查阅优先级

```
1. Context7 精准查询（最快最准）
2. 本目录的缓存文件（已沉淀的经验）
3. web_fetch 在线拉取（兜底，只拉特定页面）
```

## 文件列表

| 文件                | 覆盖内容                                     |
| ------------------- | -------------------------------------------- |
| `vue3.md`           | Vue 3 响应式、组合式 API、内置组件的高频坑点 |

| `uniapp.md`         | uni-app 条件编译、生命周期、平台差异         |
| `mongodb.md`        | 查询操作符、聚合管道、索引                   |

| `eslint-v9.md`      | ESLint v9 flat config 与 v8 的差异           |
| `typescript.md`     | tsconfig 配置、工具类型                      |
| `wot-design-uni.md` | wot-design-uni 组件用法和已知问题            |

## 追加规则

每次通过查文档解决了一个坑，用以下格式追加到对应文件末尾：

```markdown
## [简短标题]

- **来源**：[官方文档 URL]
- **结论**：[一两句话说清楚关键点]
- **日期**：YYYY-MM-DD
```

## 不该放在这里的内容

- JS 原生 API（不会变，无需缓存）
- 项目中已有 5+ 正确用例的模式（直接参考代码）
- 大段文档原文复制（只放结论和关键代码片段）
