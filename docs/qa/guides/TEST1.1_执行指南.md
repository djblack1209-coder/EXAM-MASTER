## ✅ 检查清单

### 检查点 1：应用启动日志
- [ ] 控制台出现 `App Launch`（必须有）
- [ ] 控制台出现 `App Show`（可能有，取决于页面加载时机）

### 检查点 2：登录流程日志
- [ ] 控制台出现 `[UserStore] 登录成功，用户ID: xxx`（必须有）
- [ ] 控制台出现 `[App] 静默登录成功，用户ID: xxx`（必须有）

### 检查点 3：云函数调用（关键！）
**打开 uniCloud 控制台**：
1. 登录 [uniCloud 控制台](https://unicloud.dcloud.net.cn/)
2. 进入你的服务空间
3. 点击 "云函数" → `user-center`
4. 查看 "调用日志"

**应该看到**：
- [ ] 云函数 `user-center` 被调用
- [ ] 调用参数包含 `code`（微信登录凭证）
- [ ] 调用结果返回 `{ code: 200, message: '登录成功', data: {...} }`

**如果没有调用记录**：
- ❌ 可能云对象导入失败
- ❌ 检查控制台是否有 `[UserStore] 导入云对象失败` 错误

### 检查点 4：数据库验证（关键！）
**打开 uniCloud 控制台 → 数据库**：

1. 进入 `uni-id-users` 表
2. **查看最新记录**（按 `created_at` 倒序）

**应该看到**：
- [ ] 如果用户是新用户：**新增了一条记录**
  - `openid`: 微信 OpenID（字符串）
  - `nickname`: `备考同学xxxx`（格式）
  - `avatar`: `''`（空字符串）
  - `status`: `0`（正常状态）
  - `created_at`: 当前时间戳
  - `last_login_date`: 当前时间戳
  
- [ ] 如果用户是老用户：**更新了 `last_login_date` 字段**
  - `last_login_date` 更新为当前时间戳

### 检查点 5：本地存储验证（关键！）
**打开调试器 → Storage 面板**，检查以下键：

- [ ] `EXAM_TOKEN`: 存在，值为用户ID（字符串）
- [ ] `EXAM_USER_INFO`: 存在，值为用户信息对象（JSON）
- [ ] `user_id`: 存在，值为用户ID（字符串）

**验证 `EXAM_USER_INFO` 内容**：
```json

  "_id": "用户ID",
  "id": "用户ID",
  "openid": "微信OpenID",
  "nickName": "备考同学xxxx",
  "nickname": "备考同学xxxx",
  "avatarUrl": "",
  "avatar": "",
  "created_at": 时间戳,
  "last_login_date": 时间戳

```

## ⚠️ 常见问题排查

### 问题 1：没有看到 `[UserStore] 登录成功` 日志
**可能原因**：
- 云对象导入失败
- 云函数调用失败
- 登录凭证获取失败

**排查步骤**：
1. 检查控制台是否有错误信息（红色）
2. 检查是否有 `[UserStore] 导入云对象失败` 日志
3. 检查是否有 `[UserStore] 登录异常` 日志
4. 检查网络是否正常

### 问题 2：云函数没有被调用
**可能原因**：
- uniCloud 配置错误
- 云函数未上传
- 云对象导入失败

**排查步骤**：
1. 确认 uniCloud 服务空间配置正确
2. 确认 `user-center` 云函数已上传
3. 检查控制台是否有云对象导入错误

### 问题 3：数据库没有新增记录
**可能原因**：
- 用户已存在（老用户，只更新了 `last_login_date`）
- 云函数执行失败
- 权限配置问题

**排查步骤**：
1. 检查数据库是否有相同 `openid` 的记录（可能是老用户）
2. 查看云函数调用日志，确认返回码
3. 检查数据库 Schema 权限配置

### 问题 4：本地存储中没有 Token
**可能原因**：
- 登录失败，没有保存
- Storage 写入失败（存储空间不足）

**排查步骤**：
1. 检查控制台是否有 `[StorageService] 保存失败` 日志
2. 检查登录流程是否完成
3. 尝试手动写入 Storage 测试（排除存储空间问题）
