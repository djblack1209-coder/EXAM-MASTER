# Module 7: AI聊天功能验证指南 (TEST-7.0)

## 1. 测试目标
验证 AI 聊天模块的核心功能，包括界面显示、消息发送、API 交互及错误处理。

## 2. 测试环境
- **环境**: 微信开发者工具 / 真机调试
- **账号**: 任意登录账号
- **前提**: 确保网络正常，且 `common/config.js` 中配置了有效的智谱 AI Key（或使用默认测试 Key）。

## 3. 测试用例清单

### 3.1 入口与导航
| ID | 测试项 | 操作步骤 | 预期结果 |
|---|---|---|---|
| 7.1.1 | 首页入口 | 在“刷题”页底部找到 AI 入口 | 显示“AI对话窗口”卡片 |
| 7.1.2 | 按钮文案 | 检查入口按钮文字 | 应显示“开始对话”而非“解锁” |
| 7.1.3 | 跳转逻辑 | 点击按钮 | 跳转至 `/pages/chat/chat` |

### 3.2 聊天界面 UI
| ID | 测试项 | 操作步骤 | 预期结果 |
|---|---|---|---|
| 7.2.1 | 导航栏 | 观察顶部导航栏 | 左侧返回图标，中间显示“AI助教”或好友名 |
| 7.2.2 | 初始消息 | 进入页面 | 自动显示一条 AI 的欢迎语 |
| 7.2.3 | 输入框 | 观察底部 | 输入框、表情图标、发送按钮正常显示 |

### 3.3 核心交互
| ID | 测试项 | 操作步骤 | 预期结果 |
|---|---|---|---|
| 7.3.1 | 发送文本 | 输入“你好”，点击发送 | 1. 消息上屏（绿色气泡）<br>2. 输入框清空<br>3. 出现“...”加载动画 |
| 7.3.2 | AI 回复 | 等待 API 响应 | 加载动画消失，显示 AI 回复内容（白色气泡） |
| 7.3.3 | 滚动机制 | 发送多条消息 | 页面应自动滚动到底部 |

### 3.4 异常处理
| ID | 测试项 | 操作步骤 | 预期结果 |
|---|---|---|---|
| 7.4.1 | 空内容发送 | 输入框为空时点击发送 | 无反应或提示“请输入内容” |
| 7.4.2 | 网络断开 | 断网后发送消息 | 显示红色错误提示或 Toast 提示失败 |

## 4. 自动化测试建议
可使用 `miniprogram-automator` 编写脚本模拟：
1. 点击入口
2. 输入文本
3. 截图验证

## 5. 附录：API 调试
如果 AI 不回复，请检查 `network` 面板中 `https://open.bigmodel.cn/api/paas/v4/chat/completions` 的请求状态。
