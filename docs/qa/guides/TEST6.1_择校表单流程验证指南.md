# TEST-6.1: 择校表单流程验证指南

## 📋 测试目标
验证择校智能表单的数据收集、校验和页面跳转逻辑，确保用户能够顺利完成择校信息填写并获取AI推荐结果。

## ✅ 前置条件
1. 应用已正常启动
2. 用户已登录（可选，某些功能可能需要登录）

## 🔍 测试步骤

### Step 1: 进入择校页面

1. 点击底部导航栏的 **"择校"** Tab（或通过其他入口进入 `pages/school/index.vue`）
2. 观察页面加载状态

**预期表现**：
- [ ] 页面正常加载，无错误
- [ ] 显示步骤进度条（当前在 Step 1）
- [ ] 显示表单区域

---

### Step 2: 填写基本信息表单

在 **Step 1 (背景录入)** 中填写以下信息：

1. **本科/专科毕业院校**：
   - 输入：`北京大学`（或任意测试值）
   - 验证：输入框正常显示内容，v-model 绑定正确

2. **报考专业**：
   - 输入：`计算机科学`（或任意测试值）
   - 验证：输入框正常显示内容，v-model 绑定正确

3. **报考类型**：
   - 点击选择：`学硕` 或 `专硕`
   - 验证：选中状态正确显示（高亮/激活样式），`formData.masterType` 正确更新

4. **目前学历**：
   - 点击选择：`本科全日制` 或 `专科/同等学力`
   - 验证：选中状态正确显示，`formData.degree` 正确更新

5. **英语证明**：
   - 点击选择器，选择一个选项（如：`四级`、`六级` 等）
   - 验证：选择器正常弹出，选择后显示选中的值，`formData.englishCert` 正确更新

**预期日志**：
- 可以通过 Vue DevTools 查看 `formData` 对象的实时变化
- 或通过控制台输出验证数据绑定

**预期技术表现**：
- [ ] `formData` 对象正确更新（可通过 Vue DevTools 或控制台日志验证）
- [ ] 所有输入项的数据绑定正常（v-model 工作正常）
- [ ] 选择器（picker）正常弹出和选择

## 🐛 已知问题排查

### 问题 1: 表单数据未收集
**可能原因**：
- v-model 绑定错误
- formData 初始化问题

**解决方法**：
- 检查 Vue DevTools 中的 `formData` 对象
- 检查控制台是否有绑定错误

### 问题 2: 提交后无响应
**可能原因**：
- AI API 调用失败
- 步骤切换逻辑错误

**解决方法**：
- 检查控制台错误日志
- 检查 Network 面板中的 API 请求
- 检查 `currentStep` 状态是否正确更新

### 问题 3: 推荐结果为空
**可能原因**：
- AI API 返回格式不正确
- 数据解析错误

**解决方法**：
- 检查 AI API 响应数据
- 检查数据解析逻辑
- 检查 `filteredSchools` 数组是否正确填充

**问题**：初始版本中 `uni.request` 的响应处理有问题，`statusCode` 为 `undefined`  
**原因**：直接使用 `await uni.request()` 可能不会正确返回响应对象  
**修复**：使用 Promise 包装，通过 `success`/`fail` 回调正确处理响应  
**状态**：✅ 已修复
