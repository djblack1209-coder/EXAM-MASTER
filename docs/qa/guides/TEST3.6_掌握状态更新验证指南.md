# TEST-3.6: 错题掌握状态更新验证指南

## 📋 测试目标
验证错题本中"重做此题"功能，当用户答对题目后，能够正确更新错题的掌握状态（`is_mastered: true`），并同步到云端和本地缓存。

## ✅ 前置条件
1. 错题本中至少有一条错题
2. 确保网络连接正常（用于验证云端同步）

## 🔍 测试步骤

### 步骤 1: 打开错题本页面
1. 启动微信开发者工具
2. 进入错题本页面 (`pages/mistake/index.vue`)
3. 打开控制台（Console）

### 步骤 2: 选择一道错题进行重做
1. 在错题列表中找到任意一道错题
2. 点击"重做此题"按钮
3. 观察控制台日志：
   ```
   [mistake-book] 📝 开始重做错题 - 错题ID: xxx
   ```
4. **预期结果**：
   - ✅ 错题卡片展开，显示选项列表
   - ✅ `mistake.isPracticing = true`
   - ✅ 选项可点击选择

### 步骤 3: 选择正确答案并提交
1. 查看错题的正确答案（从解析中获取，或已知答案）
2. 选择**正确答案**（例如：如果正确答案是 B，选择 B 选项）
3. 点击"提交答案"按钮
4. 观察控制台日志：
   ```
   [mistake-book] 📝 提交重做答案 - 错题ID: xxx, 选择: B, 正确答案: B, 是否正确: true
   [mistake-book] ✅ 答案正确，开始更新掌握状态 - 错题ID: xxx
   [mistake-book] ✅ 本地状态已更新 - is_mastered: true, last_practice_time: xxx
   [mistake-book] 📡 调用 storageService.updateMistakeStatus(xxx, true)
   [StorageService] 📡 开始更新错题状态 - ID: xxx, is_mastered: true
   [StorageService] ✅ 云端状态更新成功 - ID: xxx, is_mastered: true
   [StorageService] ✅ 本地缓存已同步更新
   [mistake-book] ✅ 云端状态更新成功 - source: cloud
   [mistake-book] ✅ 本地缓存已同步更新
   ```

5. **预期结果**：
   - ✅ 显示"回答正确！"的反馈
   - ✅ 错题状态更新为"已掌握"
   - ✅ 云端数据库更新成功
   - ✅ 本地缓存同步更新

### 步骤 4: 验证云端数据库更新
1. 检查云端数据库（uniCloud 控制台或数据库管理工具）
2. 查找对应错题记录
3. **预期结果**：
   - ✅ `is_mastered` 字段为 `true`
   - ✅ `last_practice_time` 字段已更新为当前时间戳

### 步骤 5: 验证本地缓存更新
1. 在控制台执行（或使用调试工具）：
   ```javascript
   const mistakes = uni.getStorageSync('mistake_book')
   const updatedMistake = mistakes.find(m => m.id === 'xxx')
   console.log('更新后的错题状态:', updatedMistake)
   ```
2. **预期结果**：
   - ✅ `is_mastered: true`
   - ✅ `isMastered: true`（兼容字段）
   - ✅ `last_practice_time` 已更新

### 步骤 6: 验证UI显示
1. 观察错题卡片
2. **预期结果**：
   - ✅ 错题不再显示"重做此题"按钮（或显示"已掌握"状态）
   - ✅ 错题卡片可能显示"已掌握"标识（如果有）

### 步骤 7: 测试错误答案场景（可选）
1. 选择另一道错题
2. 选择**错误答案**
3. 提交答案
4. **预期结果**：
   - ✅ 显示"回答错误，再试一次！"的反馈
   - ✅ 不更新掌握状态
   - ✅ 可以继续重做

## 🐛 已知问题排查

### 问题 1: 点击"重做此题"无反应
**可能原因**：
- `startPractice` 方法未正确绑定
- Vue 响应式更新问题

**解决方法**：
- 检查控制台是否有错误日志
- 确认 `@tap="startPractice(index)"` 绑定正确

### 问题 2: 提交答案后状态未更新
**可能原因**：
- `updateMistakeStatus` 调用失败
- 云端更新失败且本地更新也失败
- 错题ID为空

**解决方法**：
- 检查控制台日志，确认 `updateMistakeStatus` 是否被调用
- 检查错题是否有有效的 `id` 或 `_id`
- 验证网络连接是否正常

### 问题 3: 云端更新失败
**可能原因**：
- 网络连接问题
- 后端接口错误
- 用户未登录

**解决方法**：
- 检查网络连接
- 检查控制台错误日志
- 确认用户已登录（`EXAM_USER_ID` 存在）
- 验证降级到本地更新是否正常工作

### 问题 4: 本地缓存未更新
**可能原因**：
- `storageService.save` 调用失败
- 存储空间不足

**解决方法**：
- 检查控制台是否有存储错误
- 检查存储空间是否充足
- 验证 `save` 方法返回值
