# TEST-4.4: AI 服务超时处理验证指南

## 📋 测试目标
验证 AI 助教在请求超时（60秒）时，能够正确捕获错误、显示友好提示，且应用不崩溃。

## ✅ 前置条件
1. 确保网络连接正常（用于正常请求对比）
2. 了解如何模拟超时场景

## 🔍 测试步骤

### 方法 1: 模拟超时（推荐）

#### 操作步骤：
1. 打开微信开发者工具
2. 进入 AI 助教页面 (`pages/chat/index.vue`)
3. 打开控制台（Console）和 Network 面板
4. 在 Network 面板中，找到 AI 请求（`POST https://open.bigmodel.cn/api/paas/v4/chat/completions`）
5. 右键点击请求 → 选择"Block request URL" 或使用开发者工具的"网络节流"功能
6. 发送一条消息
7. 等待 60 秒（超时时间）
8. 观察错误处理

#### 预期日志：
```
[Chat] 发送消息: ...
[Chat] ⏱️ 请求超时（60秒）
[Chat] 📊 错误详情: { type: 'timeout', message: 'AI 解析请求超时，请稍后重试...', ... }
```

#### 预期 UI 表现：
- [ ] `isTyping` 变为 `false`（Loading 动画停止）
- [ ] 显示错误提示：`"AI 解析请求超时，请稍后重试..."`
- [ ] 不显示 AI 内容
- [ ] 应用不崩溃
- [ ] 可以继续发送新消息

---

### 方法 2: 修改超时时间（快速测试）

#### 操作步骤：

#### 预期结果：
- [ ] 1 秒后触发超时
- [ ] 显示超时错误提示
- [ ] 功能正常，可以重试

---

### 方法 3: 使用网络节流（真实场景）

#### 操作步骤：
1. 在微信开发者工具的 Network 面板中
2. 设置网络节流为 "Slow 3G" 或 "Offline"
3. 发送一条较长的消息（触发较长的处理时间）
4. 观察是否在 60 秒内超时

## 🐛 已知问题排查

### 问题 1: 超时后无错误提示
**可能原因**：
- `catch` 块未正确捕获错误
- `uni.request` 的 `fail` 回调未触发

**解决方法**：
- 检查 `catch` 块是否正确
- 检查 `fail` 回调是否正确绑定

### 问题 2: 超时时间不生效
**可能原因**：
- `timeout` 参数未正确传递
- 微信小程序平台限制

**解决方法**：
- 确认 `timeout: 60000` 已设置
- 检查平台是否支持超时设置

### 问题 3: 超时后应用崩溃
**可能原因**：
- 错误处理不完善
- 状态未正确重置

**解决方法**：
- 确保 `isTyping = false`
- 确保错误被正确捕获
