## 🔍 数据丢失的可能原因

- 这是开发环境的正常行为

- ✅ 已添加自动备份机制（下次保存时会生效）

---

### 2. 存储空间不足

**检查方法**：
```javascript
// 在控制台执行
wx.getStorageInfo({
  success: (res) => {
    const usage = (res.currentSize / res.limitSize * 100).toFixed(2);
    console.log(`存储使用率: ${usage}%`);
    console.log('当前大小:', res.currentSize, 'KB');
    console.log('限制大小:', res.limitSize, 'KB');
    console.log('已用键数:', res.keys.length);
  }
});
```

---

### 3. 误操作清空

**检查方法**：
- 查看控制台日志，搜索 `清空`、`clearAll` 等关键词
- 检查是否有清空操作的日志

## ✅ 已实施的保护措施

### 1. 自动备份机制

**下次保存时会自动创建备份**：
- 题库保存前：自动备份到 `v30_bank_backup`
- 文件记录保存前：自动备份到 `imported_files_backup`

### 2. 清空前备份机制

**清空操作前会创建备份**：
- 清空题库前：自动备份到 `v30_bank_backup_before_clear`
- 即使误操作，也能从备份恢复

### 3. 自动恢复机制

**数据丢失时自动尝试恢复**：
- 刷题中心页面：自动从多个备份源恢复
- 文件管理页面：自动从备份恢复

### 步骤 2：验证备份是否创建

上传完成后，在控制台检查：

```javascript
// 检查题库备份
const bankBackup = wx.getStorageSync('v30_bank_backup');
if (bankBackup) {
  const restored = JSON.parse(bankBackup);
  console.log('✅ 题库备份存在:', restored.length, '道题');
} else {
  console.log('❌ 题库备份不存在');
}

// 检查文件备份
const filesBackup = wx.getStorageSync('imported_files_backup');
if (filesBackup) {
  const restored = JSON.parse(filesBackup);
  console.log('✅ 文件备份存在:', restored.length, '个文件');
} else {
  console.log('❌ 文件备份不存在');
}
```

---

### 步骤 3：检查存储空间

如果存储空间不足，备份可能失败：

```javascript
wx.getStorageInfo({
  success: (res) => {
    const usage = (res.currentSize / res.limitSize * 100).toFixed(2);
    if (usage > 80) {
      console.warn('⚠️ 存储空间不足:', usage + '%');
      console.log('建议清理其他数据');
    } else {
      console.log('✅ 存储空间充足:', usage + '%');
    }
  }
});
```

## 🔧 预防措施

### 1. 避免频繁重新编译

**建议**：
- 开发时，尽量避免频繁重新编译
- 如果需要重新编译，先检查数据是否已保存
- 重要数据建议上传到云端（未来功能）

---

### 2. 定期检查存储空间

**建议**：
- 定期检查存储使用率
- 如果超过 80%，考虑清理不必要的数据
- 或者考虑使用云端存储

---

### 3. 重要数据云端备份

**未来改进**：
