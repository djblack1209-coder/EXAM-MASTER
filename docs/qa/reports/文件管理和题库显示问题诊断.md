# 文件管理和题库显示问题诊断

## 📋 问题描述

用户反馈：
- **文件管理中不显示**：刚刚上传的资料在文件管理中不显示
- **题库显示空空如也**：题库也显示空空如也

---

## 🔍 问题分析

### 1. 数据存储方式不一致

**发现的问题**：
- `import-data.vue` 使用 `uni.setStorageSync` 保存数据
- `practice/index.vue` 使用 `storageService.get` 读取数据
- `file-manager.vue` 使用 `uni.getStorageSync` 读取数据

**可能的影响**：
- 如果 `storageService` 和 `uni.getStorageSync` 使用不同的存储机制，可能导致数据不一致

### 2. 文件记录保存逻辑

**代码位置**：`src/pages/practice/import-data.vue`

**保存时机**：
- 在 `handleUpload()` 方法中调用 `saveUploadRecord()`
- 文件记录保存到 `imported_files` 存储键

**可能的问题**：
- 保存时可能出错但没有错误提示
- 保存后没有验证是否成功

### 3. 题库保存逻辑

**代码位置**：`src/pages/practice/import-data.vue`

**保存时机**：
- 在 `handleAIResult()` 方法中保存题库
- 题库保存到 `v30_bank` 存储键

**可能的问题**：
- AI 返回结果解析失败
- 保存时可能出错但没有错误提示
- 保存后没有验证是否成功

## 🔧 排查方法

### 如果文件管理页面不显示文件

**步骤 1**：检查控制台日志
- 查看是否有 `[文件管理]` 标记的日志
- 检查读取到的文件数量是否为 0

**步骤 2**：手动检查存储
```javascript
// 在控制台执行
const files = uni.getStorageSync('imported_files') || [];
console.log('文件列表:', files);
```

**步骤 3**：检查文件记录保存
- 查看导入资料页面的控制台日志
- 确认 `saveUploadRecord` 是否被调用
- 确认保存是否成功
