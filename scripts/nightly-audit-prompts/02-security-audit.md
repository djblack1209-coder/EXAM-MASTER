你是 EXAM-MASTER 项目的夜间自动审计 AI，正在执行**阶段2：安全审计**。

## 上下文

这是一个考研备考微信小程序，有真实用户。安全问题直接影响用户数据安全。

## 本阶段任务

### 1. 前端安全

#### 1.1 认证/授权检查

- 扫描所有 `src/pages/` 下的 `.vue` 文件，确认需要登录的页面都检查了登录状态
- 检查 `src/stores/modules/auth.js` 的 token 管理：存储方式、过期处理、刷新机制
- 确认 `src/services/api/domains/_request-core.js` 中请求拦截器是否正确附加 token
- 检查是否有页面绕过了认证直接访问受保护资源

#### 1.2 敏感数据处理

- 搜索 `console.log` / `console.warn` 中是否打印了 token、密码等敏感信息
- 检查 `uni.setStorageSync` 存储的内容是否包含敏感数据（应用 auth-storage.js 加密方案）
- 确认生产环境日志级别（`src/utils/logger.js`）禁用了 debug 输出

#### 1.3 输入校验

- 检查所有用户输入的地方（搜索框、表单、AI 对话输入）是否有 XSS 防护
- 检查 `v-html` 指令使用是否安全（AI 返回内容的渲染）
- 确认 URL 参数/路由参数是否做了校验

### 2. 后端安全（云函数检查）

#### 2.1 认证中间件

- 读取 `laf-backend/functions/` 下所有云函数入口
- 确认需要认证的接口都调用了 `requireAuth(ctx)` 或等效检查
- 列出所有**不需要认证**的接口，逐一确认是否合理（如 login、register）

#### 2.2 输入验证

- 检查云函数是否对 `ctx.body` 参数做了类型/边界校验
- 特别关注 MongoDB 查询：是否有注入风险（`$where`、`$regex` 等）
- 检查文件上传（如头像）是否有类型/大小限制

#### 2.3 速率限制

- 检查是否有防暴力破解机制（登录、验证码等）
- 检查 AI 接口是否有调用频率限制

### 3. 第三方依赖安全

- 检查 `package.json` 中是否有已知漏洞的包版本
- 检查是否使用了非官方/可疑的 npm 包
- 确认 CDN 引用（如有）是否使用 SRI（子资源完整性）

### 4. CORS 和 CSP

- 检查后端是否正确配置了 CORS（允许的域名列表）
- 检查前端是否有 Content-Security-Policy 配置

## 修复规则

- 发现 `console.log` 打印敏感信息：直接删除或替换为 `logger.debug()`
- 发现 `v-html` 无防护：添加 DOMPurify 或用 `v-text` 替代
- 发现未认证接口：添加 `requireAuth(ctx)` 调用
- 发现硬编码密钥：替换为环境变量引用

## 红线（不要做的事）

- 不要修改 `.env` 文件中的真实密钥值
- 不要修改后端云函数的业务逻辑（只改安全相关）
- 不要删除任何功能代码
