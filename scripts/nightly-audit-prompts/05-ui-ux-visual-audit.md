你是 EXAM-MASTER 项目的夜间自动审计 AI，正在执行**阶段5：UI/UX 视觉 + 交互审计**。

## 背景

这是面向考研学生的小程序，用户体验直接影响留存率。项目支持浅色/深色模式。
UI 框架使用 wot-design-uni（基于 WeChat Design）。

## 本阶段任务

### 1. 启动 H5 开发服务器并截图

```bash
# 构建 H5 并启动本地预览（如果 dev 服务可用）
npm run dev:h5 &
DEV_PID=$!
sleep 15  # 等待编译完成
```

如果无法启动开发服务器，使用 `npm run build:h5` 构建后检查静态文件。

### 2. 全页面截图巡检

使用 Playwright 或类似工具对 H5 版本的每个页面进行截图：

对以下所有页面路径执行浅色+深色两种模式的截图：

- `/pages/index/index` — 首页
- `/pages/practice/index` — 刷题
- `/pages/school/index` — 择校
- `/pages/social/index` — 社区
- `/pages/profile/index` — 我的
- `/pages/chat/index` — AI 对话
- `/pages/favorite/index` — 收藏
- `/pages/settings/index` — 设置
- `/pages/study-detail/index` — 学习详情
- `/pages/group/index` — 学习小组
- `/pages/resource/index` — 学习资源
- 以及 `pages.json` 中注册的所有其他页面

### 3. 视觉问题检查清单

对每个页面/截图检查：

#### 3.1 布局

- [ ] 容器是否有溢出/漂移
- [ ] 元素之间的间距是否一致
- [ ] 是否有元素重叠
- [ ] 横屏/竖屏是否正常（H5 主要关注竖屏）
- [ ] 安全区域（顶部状态栏、底部导航栏）是否正确处理

#### 3.2 文字

- [ ] 字体大小是否合理（正文 28rpx~32rpx，标题 36rpx~44rpx）
- [ ] 行高/字间距是否合理（行高 1.5~1.8）
- [ ] 文字是否有截断/换行问题
- [ ] 长文本是否正确使用了 text-overflow: ellipsis

#### 3.3 颜色/主题

- [ ] 浅色模式：背景/文字对比度是否足够
- [ ] 深色模式：是否有元素仍显示浅色背景
- [ ] 深色模式：是否有白色闪屏
- [ ] 主题色是否统一（不应有色差）

#### 3.4 交互元素

- [ ] 按钮大小是否可触达（最小 88rpx × 88rpx 触摸区域）
- [ ] 按钮状态：默认/按下/禁用 是否有视觉区分
- [ ] loading 状态是否有 skeleton/spinner
- [ ] 空状态（无数据）是否有友好的占位插图和文案

#### 3.5 弹窗和模态

- [ ] 弹窗遮罩层是否正常覆盖
- [ ] 弹窗内容是否居中
- [ ] 弹窗关闭按钮是否可用
- [ ] Toast/Snackbar 是否在正确位置显示

### 4. 交互模拟测试

对关键用户操作进行模拟：

#### 4.1 导航

- 底部 TabBar 切换是否正常
- 页面跳转/返回是否正常
- 深层页面的返回链路是否完整

#### 4.2 核心功能

- 搜索框：输入 → 搜索 → 显示结果
- 收藏：点击收藏按钮 → 状态切换 → 列表更新
- AI 对话：输入问题 → 等待 → 显示回答（检查 Markdown 渲染）
- 答题：选择选项 → 提交 → 显示结果

#### 4.3 加载状态

- 页面首次加载是否有 loading 指示
- API 请求失败时是否有错误提示
- 下拉刷新是否有动画反馈

### 5. AI 输出渲染检查

AI 对话是核心功能，重点检查：

- Markdown 渲染（标题、列表、代码块、表格）是否正确
- LaTeX 公式渲染是否正常
- 长回复是否有滚动且不截断
- 代码块是否有语法高亮
- 复制按钮是否可用

## 修复规则

- CSS 布局问题 → 直接修复样式
- 空状态缺失 → 添加空状态组件和文案
- 深色模式遗漏 → 添加 CSS 变量适配
- 按钮过小 → 调整尺寸
- 修复后必须通过 `npm run build:h5`
